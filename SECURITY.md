# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.1.x   | :white_check_mark: |
| 2.0.x   | :white_check_mark: |
| < 2.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Nebula-Writer seriously. If you find a vulnerability, please report it via one of the following methods:

1. **GitHub Issue**: For non-sensitive issues, please open a standard issue.
2. **Direct Contact**: For sensitive security vulnerabilities, please email the maintainer directly at [sagar0163@example.com] (placeholder).

Please do not disclose the vulnerability publicly until we have had a chance to fix it.

## Security Practices

- We use environment variables for sensitive API keys.
- We regularly update our dependencies to patch known vulnerabilities.
- We run automated security scans as part of our CI/CD pipeline.
